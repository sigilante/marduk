# plan-kernel tutorials

Notebook-based walkthroughs of PLAN and Plan Asm. Each lesson is a
self-contained `.ipynb` that runs in the `plan-kernel` Jupyter kernel.

## Running

Install the kernel once per environment:

```bash
pip install -e ../../marduk -e ..        # marduk runtime + plan-kernel
python -m plan_kernel install            # registers the kernelspec
```

Then open a tutorial in JupyterLab or notebook:

```bash
jupyter lab tutorials/01-numbers-and-pins.ipynb
```

Re-run all cells (Run → Run All Cells) to evaluate against your local
kernel.

## Index

| File                                | Topic                                                                                              |
|-------------------------------------|----------------------------------------------------------------------------------------------------|
| `01-numbers-and-pins.ipynb`         | Bare nats, BPLAN arithmetic primitives, comparison ops, `#pin`.                                    |
| `02-laws.ipynb`                     | Defining laws with `#law`, naming via `#bind`, identity / K / composition, `#pin`-wrapping a law.  |
| `03-elim-and-cases.ipynb`           | `Elim` six-arm pattern matcher, `Case2..Case16` shortcuts, Church-style booleans.                  |
| `04-recursion.ipynb`                | Self-recursive laws (factorial, fibonacci, accumulator-sum), depth caveat.                         |

The lessons are sequential — each one assumes the previous.

## Authoring

Each lesson is generated by [`_build_lessons.py`](_build_lessons.py),
which holds a Python list of `(kind, body)` cell descriptors per
lesson. The script runs every code cell through `PlanKernelEvaluator`
to capture real outputs, so the committed notebook always reflects
what the kernel produces.

To edit a lesson:

1. Modify the `LESSON_NN` list in `_build_lessons.py`.
2. Run `python tutorials/_build_lessons.py` to regenerate the notebook
   (or `python tutorials/_build_lessons.py 02 03` to rebuild a subset).
3. Verify with `python -m pytest tests/test_tutorials.py`.

The test suite catches output drift: if the kernel's renderer changes
a cell's output, the tutorial test fails until you regenerate the
notebook.
